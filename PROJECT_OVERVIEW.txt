🎯 CORE ALGORITHM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  GRAPH STRUCTURE
  ├─ Nodes: Tokens (XLM, USDC, USDT, ETH, BTC)
  ├─ Edges: AMM pools with reserves and fees
  └─ Bidirectional trading support

  DIJKSTRA ALGORITHM
  ├─ Maximizes output amount (not minimizes distance)
  ├─ BFS-style traversal with depth constraint
  ├─ MAX_HOPS = 3 (for gas efficiency)
  ├─ Time Complexity: O(T² × E) ≈ O(50)
  └─ Space Complexity: O(T + P)

  CONSTANT PRODUCT FORMULA (CPF)
  ├─ Formula: output = (inputAfterFee × reserveOut) / (reserveIn + inputAfterFee)
  ├─ Fee Deduction: inputAfterFee = amountIn × (10000 - feeInBasisPoints) / 10000
  ├─ Invariant: x × y = k (reserve product must increase)
  └─ BigInt Precision: No floating-point errors
